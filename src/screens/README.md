# Screens

This directory contains full-page screen components that compose together design system components and business logic.

## Structure

- **/dashboards** - Dashboard screens for different user roles
- **/modules** - Full module screens (HR, Finance, Inventory, etc.)
- **/auth** - Authentication screens (Login, Register, etc.)
- **/reports** - Report screens
- **/settings** - Settings and configuration screens

## Naming Convention

- Use PascalCase for screen component files
- Suffix with "Screen" for clarity (e.g., `DashboardScreen.tsx`)
- Keep screens focused on layout and orchestration, not business logic
